Workflow Intake System

Used Python, Flask, Javascript, Heidi SQL, CSS, and HTML to create an application where users could securely submit and track requests for automation. The teams could prioritize, and follow through on requests.

Front-end and back-end validation were used for the requestor form to ensure accurate and appropriate inputs into the database for storage. This data was then stored, retrieved, and altered so that the user could track and edit their requests. The application used LDAP group permissions so users had access controls. Only admins could see all the requests submitted, to assign the request to a respective system. In contrast, a normal user could only see their request.
